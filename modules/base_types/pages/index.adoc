= {spec_title} [.spec-status.spec-status-stable]#{spec_status}#

Core identifier and reference types shared across the openEHR specification stack: object identifiers, version identifiers, terminology codes, and typed object references. Object identifiers support both UUID-based and ISO OID-based schemes, while version identifiers encode three-way version logic for branch-aware versioning. Every versioned EHR object, archetype, and terminology reference in the platform is identified using types defined in this specification.

== Purpose

This document describes the openEHR Base Types, a collection of general types used by other openEHR specifications.

The intended audience includes:

* Standards bodies producing health informatics standards;
* Research groups using openEHR, {iso_13606}[ISO 13606^], archetypes and related technologies;
* The open source healthcare community;
* Solution vendors.
