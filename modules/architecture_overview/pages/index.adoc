= {spec_title} [.spec-status.spec-status-stable]#{spec_status}#

Read-first introduction to the openEHR specification stack. Covers the multi-level modelling paradigm, principal components, relationships between specifications, and the global design philosophy of the platform. It provides the essential conceptual grounding for understanding how the Reference Model, Archetype Model, Service Model, and implementation technology specifications relate to each other. Readers approaching openEHR for the first time should start here before exploring the individual component specifications.

== Purpose

This document provides an overview of the openEHR architecture in terms of a model overview, key global semantics, deployment and integration architectures, relationship to published standards, and finally the approach to building Implementation Technology Specifications (ITSs). Semantics specific to each information, archetype and service model are described in the relevant specification.

The intended audience includes:

* Standards bodies producing health informatics standards;
* Academic groups using openEHR;
* The open source healthcare community;
* Solution vendors.

This document is the key technical overview of openEHR, and should be read before all other technical documents.
